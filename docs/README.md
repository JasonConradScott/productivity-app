# Documentation

Product and domain documentation for the productivity app (personal, goal-ladder, energy-aware).

| Document | Purpose |
|----------|---------|
| [VISION.md](./VISION.md) | Product anchor — who, what a good day means, principles, pains, non-goals. **Grows** as domains mature. |
| [DOMAINS.md](./DOMAINS.md) | Working map of capability domains — problems, v1 ideas, scope, dependencies. |

Start in **DOMAINS.md** to flesh out each area; pull stable one-liners into **VISION.md** as you go. Key ideas: **Short list** (max 3 tasks, **focus** highlighted); **mental focus demand** (**deep** vs **flexible**) with **soft** morning vs later-day fit; **flexible-length** work blocks (smaller when tired); **optional** block **duration** tracking and **Reflection** trends for **motivation**; **vision board** with **optional** **goal progress look-back**; **Pomodoro-like** rhythm; **between-blocks ritual**; **exercise** logging and trends; **three typed breaks**; **Rituals** (*Atomic Habits*–compatible).
