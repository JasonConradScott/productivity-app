# Documentation

Product and domain documentation for the productivity app (personal, **horizon ladder**, energy-aware).

| Document | Purpose |
|----------|---------|
| [VISION.md](./VISION.md) | Product anchor — who, what a good day means, principles, pains, non-goals. **Grows** as domains mature. |
| [DOMAINS.md](./DOMAINS.md) | Working map of capability domains — problems, v1 ideas, scope, dependencies. **Horizon ladder** (§5): seven rungs, one list each, rank; **Life tags** (unique on Life items, reused on lower rungs). Includes **UI (product surface)** (ladder nav, fast capture, inbox triage, drag-and-drop). |
| [DEFINITIONS.md](./DEFINITIONS.md) | **Canonical terms**, naming preferences, **glossary**, and **doc/DB workflow** for consistency. |

Start in **DOMAINS.md** to flesh out each area; pull stable one-liners into **VISION.md** as you go. Key ideas: **horizon ladder** (Life → Daily, **one list per rung**, **priority rank**, **Life tags** defined on Life rung, **added** to lower rungs); **minimal inbox** (fast capture, drag-drop; **inbox triage** work block assigns rung, rank, tags); **short list** (max 3, **focus** highlighted, usually **Daily**); **mental focus demand**; **flexible-length** work blocks; **optional** block tracking; **vision board** + **goal progress look-back**; **daily life maintenance** list (resets daily, **between-blocks** UI); **later** — **achievements** / **gamification** (e.g. Fitbit-style), ladder-linked and opt-in; **Pomodoro-like** rhythm; **between-blocks ritual**; **exercise** logging; **three typed breaks**; **Rituals** (*Atomic Habits*–compatible).
